# Phase 1: 深掘り（Discovery）フェーズ定義

## ゴール
案件概要（brief）から、具体的で実装可能な要件を明確化し、プロジェクトのスコープとリスクを定義する。

## 参加ロール
- **PM（プロジェクトマネージャー）**: 議論の進行管理、ゲート判定
- **BA（ビジネスアナリスト）**: 要件の深掘り、ユーザーストーリー作成
- **Architect（アーキテクト）**: 技術的実現可能性の評価

## 議論の進め方
1. BAが案件概要を分析し、初期の要件候補を提示する
2. Architectが技術的な観点から実現可能性を評価する
3. PMが議論を整理し、不足している要件を指摘する
4. チーム全体で要件の優先順位を合議する
5. PMがゲート通過を判断する

## ゲート基準（Phase 1 → Phase 2 進行条件）
以下がすべて満たされた場合、PMはゲート通過を宣言する：
- [ ] 機能要件がユーザーストーリー形式で列挙されている
- [ ] 非機能要件（性能、セキュリティ等）が定義されている
- [ ] スコープ（やること / やらないこと）が明確である
- [ ] 主要なリスクが洗い出されている
- [ ] 技術的に実現不可能な要件がないことが確認されている

## 成果物
- `requirements.md` — 要件定義書
- `risks.md` — リスク一覧

## 入力
- `brief.md` — 案件概要（ユーザー入力）
